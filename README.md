# Episode Bets

Мини-сайт для ставок игровой валютой на события одной серии сериала.

## Как это работает

- Игрок начинает со 100 токенами.
- Он нажимает `Старт` примерно одновременно с началом просмотра серии.
- Сайт ведет внутренний таймлайн серии и открывает ставки по времени из `data/episode-demo.json`.
- У каждой ставки есть окно открытия, окно закрытия, время расчета, варианты исхода и коэффициенты.
- Игрок может сделать одну ставку на один рынок.
- Если выбранный исход совпадает с `winningOptionId`, баланс пополняется на `ставка * коэффициент`.
- Прогресс игрока сохраняется в браузере для выбранной серии.
- До старта серии может быть отдельный блок предсерийных ставок. При нажатии `Старт` сайт напоминает поставить на них.

## Почему это подходит для GitHub Pages

Проект полностью статический: `index.html`, `styles.css`, `app.js` и JSON с данными. Сервер, база данных и авторизация в первой версии не нужны. GitHub Pages сможет отдать эти файлы как обычный сайт.

Ограничения первой версии:

- баланс и ставки хранятся только в браузере пользователя;
- пользователь может сбросить игру;
- нет защиты от просмотра JSON с правильными ответами;
- нет общей таблицы лидеров.

Это нормально для прототипа и одиночного фан-режима. Для честной соревновательной игры понадобится сервер или serverless-бэкенд.

## Локальный запуск

Если рядом с `index.html` есть JSON-файл, сайт лучше открывать через локальный веб-сервер:

```bash
start-site.cmd
```

Или вручную:

```bash
node dev-server.mjs
```

После запуска откройте `http://127.0.0.1:4173/`.

Не открывайте `index.html` напрямую через `file://`: браузер может заблокировать загрузку JSON с данными серии.

## Деплой на GitHub Pages

Проект можно публиковать прямо из корня репозитория, без сборки.

1. Создайте репозиторий на GitHub.
2. Загрузите в него файлы проекта.
3. В настройках репозитория откройте `Settings` -> `Pages`.
4. В `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку `main` и папку `/root`.
6. Сохраните настройки и дождитесь ссылки Pages.

Файл `.nojekyll` уже добавлен, чтобы GitHub Pages отдавал статические файлы как есть.

## Формат данных серии

Основной файл: `data/episode-demo.json`.

Список доступных серий хранится в `data/episodes.json`. Чтобы добавить новую серию, создайте новый JSON-файл в `data` и добавьте его в этот список.

Важные поля рынка:

- `betFormat` - формат ставки: `choice` или `text`;
- `marketType` - тип ставки: `quick30`, `minute`, `long30`, `final`;
- `opensAt` - секунда серии, когда ставка появляется;
- `closesAt` - секунда серии, когда ставка закрывается;
- `resolveAt` - секунда серии, когда ставка считается;
- `options` - варианты ответа с коэффициентами;
- `winningOptionId` - правильный исход.

Рекламные блоки задаются в `adBreaks`:

- `startsAt` - секунда начала рекламы;
- `endsAt` - секунда конца рекламы;
- `label` - подпись на таймере, например `Реклама`.

Когда таймлайн попадает в рекламный блок, рядом с таймером появляется подпись `Реклама`, а кнопка `Пропустить рекламу` переносит таймер к концу текущего рекламного блока. Кнопки `-10 сек` и `+10 сек` помогают вручную подстроиться под плеер.

Заставка задается в `introBreaks` по тому же принципу. Во время заставки появляется подпись `Заставка` и кнопка `Пропустить заставку`.

Предсерийные ставки задаются в `preStartMarkets`. Они используют тот же формат, что и обычные рынки, но закрываются сразу после старта просмотра.

Форматы ставок:

- `choice` - игрок выбирает один вариант из списка. У каждого варианта свой коэффициент.
- `text` - игрок сам вводит ответ. Сайт сравнивает его с `correctAnswers` из базы серии и применяет общий `textOdds`.

Текущие типы ставок:

- `quick30` - короткое окно на 30 секунд;
- `minute` - окно на 1 минуту;
- `long30` - длинное окно примерно на 30 минут;
- `final` - прогноз до финальной развязки.

## Режим автора

Режим автора скрыт от обычного игрока. Чтобы открыть рабочую форму для сборки рынков ставок, добавьте к адресу сайта `?admin=1`.

Пример локально:

```text
http://127.0.0.1:4173/?admin=1
```

На GitHub Pages:

```text
https://USER.github.io/REPOSITORY/?admin=1
```

Это не настоящая серверная авторизация, а скрытый авторский режим для статического фан-прототипа. Если понадобится защита по логину, нужен сервер или serverless-бэкенд.

В авторском режиме:

- перемотайте таймлайн на нужный момент серии;
- выберите формат ставки: выбор из списка или открытый ответ;
- выберите тип ставки: 30 секунд, 1 минута, 30 минут или финальный прогноз;
- нажмите `Сейчас` рядом с полем открытия, закрытия или расчета;
- добавьте вопрос, варианты исхода и коэффициенты;
- укажите `ID верного исхода`;
- нажмите `Добавить в черновик`;
- после разметки скопируйте или скачайте JSON.

Черновик хранится в `localStorage` браузера. Он не публикуется сам: после экспорта нужно заменить или добавить JSON-файл в папке `data`.

Черновики и прогресс хранятся отдельно для каждой серии.

Формат вариантов в авторской форме:

```text
yes | Да | 1.8
no | Нет | 2.2
```

Коэффициент может быть `1`, если ставка нужна как фан-событие без выигрыша сверху.

Для открытого ответа укажите общий коэффициент и список правильных ответов, по одному в строке. Пользователь увидит поле ввода и попытается угадать ответ сам.

## Как получить диалоги и сюжетные линии из видео

Для авторской разметки лучше сначала получить транскрипт с таймкодами, а потом вручную превратить его в рынки ставок.

Варианты:

- Azure AI Video Indexer: загрузка видео, транскрипт, сцены и видео-инсайты. Удобно, если нужен не только текст, но и разбор видео.
- Google Cloud Video Intelligence / Speech-to-Text: транскрипция речи из видео или аудио.
- OpenAI Speech-to-Text: лучше сначала извлечь аудио из видео, затем отправить аудиофайл на транскрипцию.

Практический процесс:

1. Извлечь аудио из серии.
2. Получить транскрипт с таймкодами.
3. Разбить транскрипт на сцены и поворотные моменты.
4. Выписать потенциальные ставки в авторском режиме сайта.
5. После проверки скачать JSON и добавить его в `data`.

Не загружайте чужие или приватные видео в случайные сервисы без прав на обработку контента. Для фан-прототипа безопаснее работать с собственными материалами, публичными тестовыми видео или локальными инструментами.

## Что подготовить для реальной серии

1. Разметить таймкоды сцен.
2. Выписать потенциальные ставки.
3. Для каждой ставки указать окно активности и момент расчета.
4. Добавить правильный исход после просмотра серии.
5. Проверить, что короткие ставки не появляются слишком поздно для живого просмотра.
